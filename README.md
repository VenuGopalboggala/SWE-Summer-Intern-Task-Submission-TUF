# 📅 Interactive Wall Calendar Component

A modern, responsive **Wall Calendar UI** built using **Next.js (React)** and **Tailwind CSS**, inspired by a physical wall calendar design.

This project demonstrates strong frontend engineering skills including **component architecture, state management, UI/UX design, and responsiveness**.
---

## 📂 GitHub Repository

👉 https://github.com/VenuGopalboggala/SWE-Summer-Intern-Task-Submission-TUF

---

## 🎯 Features

### 📅 Wall Calendar UI

* Clean layout inspired by a physical wall calendar
* Hero image section for visual appeal
* Balanced layout between calendar and notes

### 📆 Date Range Selection

* Select **start and end dates**
* Highlights:

  * Start date
  * End date
  * Range in between
* Smooth visual feedback

### 📝 Notes Section

* Add notes for selected date range
* Data stored using **localStorage**
* Persistent across refresh

### 🔄 Month Navigation

* Navigate between months using arrow buttons
* Dynamic calendar rendering

### 🎞 Animation

* Smooth **calendar flip animation** using Framer Motion

### 📱 Fully Responsive

* Desktop → side-by-side layout
* Mobile → stacked layout
* Touch-friendly interactions

---

## 🛠 Tech Stack

* **Framework:** Next.js (React)
* **Styling:** Tailwind CSS
* **State Management:** React Hooks
* **Date Handling:** date-fns
* **Animations:** Framer Motion

## 📦 Installation & Setup

Clone the repository:

```bash
git clone https://github.com/VenuGopalboggala/SWE-Summer-Intern-Task-Submission-TUF.git
cd SWE-Summer-Intern-Task-Submission-TUF
```

Install dependencies:

```bash
npm install
```

Run the development server:

```bash
npm run dev
```

Open in browser:

```
http://localhost:3000
```

## 🧠 Design & Engineering Decisions

* **Component-Based Architecture**

  * Separated logic into reusable components:

    * Calendar
    * DateRangePicker
    * NotesPanel
    * HeroImage

* **Local Storage for Persistence**

  * Keeps the app frontend-only as required
  * No backend dependency

* **Responsive Design**

  * Mobile-first approach using Tailwind

* **User Experience Focus**

  * Clear visual states
  * Accessible color contrast
  * Smooth transitions
